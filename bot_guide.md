# 🏆 Maple World Discord Bot 가이드 & 문제 해결 문서

이 문서는 초대 추적 및 입장 역할 부여 봇의 상세 사용 방법, 작동 원리, 그리고 봇 운영 중 발생할 수 있는 여러 오류 상황에 대한 해결 방법을 담은 가이드라인입니다.

---

## 📌 1. 주요 기능 개요

이 봇은 디스코드 서버 내의 초대 링크 생성과 사용 횟수를 모니터링하여 **누가 누구를 통해 가입했는지 추적**하고, 가입 시 **버튼을 클릭하여 지정된 역할을 부여받는 기능** 및 **실시간 입장 로그 기록**을 지원합니다.

---

## 🛠️ 2. 슬래시 명령어 사용 가이드
모든 슬래시 명령어는 기본적으로 **서버 관리자(Administrator) 권한**을 보유한 유저만 실행 가능하도록 제한되어 있습니다.

### ① `/서버입장생성`
* **설명**: 신규 유저가 서버 입장 시 클릭하여 역할을 지급받을 수 있는 안내 메시지(Embed)와 버튼을 생성합니다.
* **매개변수 (Arguments)**:
  * `role` (필수): 유저가 버튼을 눌렀을 때 자동으로 지급할 디스코드 역할입니다.
  * `channel` (선택): 안내 임베드와 버튼이 전송될 텍스트 채널입니다. 지정하지 않을 경우 명령어를 전송한 현재 채널에 생성됩니다.

### ② `/입장로그설정`
* **설명**: 신규 유저가 초대 링크로 들어왔을 때 출력되는 웰컴 메시지(입장 로그) 전송 기능을 키거나 끄고, 출력할 전용 채널을 등록합니다.
* **매개변수 (Arguments)**:
  * `enabled` (필수): 로그 기록 활성화 여부를 선택합니다 (`True`/`False`).
  * `channel` (선택): 로그 메시지가 전송될 채널을 지정합니다. 지정하지 않으면 `.env`의 기본 로깅 채널 또는 서버 시스템 채널에 출력됩니다.

### ③ `/초대순위`
* **설명**: 서버 내부의 누적 초대 성공 횟수를 산출하여 1위부터 최대 10위까지 순위를 표기하는 리더보드 임베드를 출력합니다. (일반 사용자의 접근 권한은 차단되어 있으며 관리자만 조회 가능합니다)
* **산출 공식**: `초대 가입자 수 (join_count) - 중도 퇴장자 수 (leave_count)`

---

## ⚙️ 3. 관리자 전용 수동 명령어 (채팅창 입력)
봇의 슬래시 명령어 동기화에 문제가 있을 경우 사용 가능한 특수 접두사(`!!`) 명령어입니다. **서버 관리자만 사용 가능**합니다.

| 명령어 | 동작 설명 |
| :--- | :--- |
| **`!!sync`** | 현재 서버에 한해 슬래시 커맨드를 디스코드 API 서버에 강제 재등록합니다. |
| **`!!clear`** | 글로벌 서버 및 현재 서버의 동기화 캐시를 모두 제거합니다. |

---

## 🚨 4. 대표적인 오류 발생 시 해결 방법 (Troubleshooting)

### Q1. 슬래시 명령어를 입력하려는데 목록에 봇의 명령어가 보이지 않습니다.
* **원인**: 디스코드 클라이언트의 API 명령어 동기화 지연 또는 캐싱 문제입니다.
* **해결 방법**:
  1. 봇이 정상 기동되었는지 터미널 창의 로그를 점검합니다.
  2. 서버 채팅창에 **`!!sync`**를 전송하여 동기화를 명시적으로 실행시킵니다.
  3. 디스코드 프로그램에서 **`Ctrl + R`** (모바일은 앱 완전 종료 후 재시작)을 눌러 화면을 새로고침하여 캐시를 강제 갱신합니다.

### Q2. 슬래시 명령어가 똑같은 디자인으로 2개씩 중복되어 노출됩니다.
* **원인**: 디스코드 글로벌(Global) 명령어 영역과 개별 서버(Guild) 명령어 영역 양쪽 모두에 커맨드 스키마가 등록되어 충돌했기 때문입니다.
* **해결 방법**:
  1. 채팅창에 **`!!clear`**를 입력하여 기존의 중복된 길드/글로벌 커맨드를 청소합니다.
  2. **[필수]** 메모리 명령어 구조가 삭제되므로 **봇 프로그램 프로세스를 완전히 껐다가 다시 켜서 재기동**합니다.
  3. 디스코드 화면에서 **`Ctrl + R`**을 눌러 클라이언트를 새로고침합니다.
  4. 봇이 켜진 것을 확인한 뒤 **`!!sync`**를 입력하여 1개의 커맨드 세트만 깨끗하게 재동기화시킵니다.

### Q3. `!!sync`를 실행했는데 "0개의 명령어를 동기화 완료했습니다" 라며 명령어가 사라졌습니다.
* **원인**: `!!clear` 실행으로 봇의 메모리에 탑재된 트리까지 함께 초기화되어 발생한 상태입니다.
* **해결 방법**:
  * 봇 프로그램 구동 콘솔에서 프로세스를 중단한 뒤 **재부팅**을 진행하면 코드 내의 슬래시 커맨드 트리가 복원됩니다. 이후 재부팅된 상태에서 `!!sync`를 다시 시도하십시오.

### Q4. 유저가 '서버 입장' 버튼을 클릭했으나 "역할 부여 중 오류가 발생했습니다" 또는 "봇에게 역할 부여 권한이 없습니다" 에러가 발생합니다.
* **원인**: 디스코드 역할 구조상 봇의 고유 역할 순위(높이)보다 유저에게 지급하려는 대상 역할의 순위가 더 높거나, 봇에게 '역할 관리하기(Manage Roles)' 권한이 부여되지 않았기 때문입니다.
* **해결 방법**:
  1. 디스코드 서버 설정 > 역할 관리 메뉴로 이동합니다.
  2. **봇의 전용 역할(봇 이름과 동일한 역할)을 드래그하여 유저에게 줄 대상 역할보다 위쪽 순위에 위치**시키십시오.
  3. 봇의 역할 권한에 '역할 관리하기' 권한이 체크되어 있는지 확인하십시오.

### Q5. 유저가 입장했음에도 웰컴 채널에 초대 추적 및 환영 메시지가 전송되지 않습니다.
* **원인**: 봇에게 대상 텍스트 채널에 대한 '메시지 전송', '링크 첨부(Embed)' 권한이 누락되었거나, 봇이 초대 정보(Uses 횟수 변동)를 분석하지 못하는 상태일 수 있습니다. (서버 보안 설정 상 '서버 멤버 목록 보기' 권한 필요)
* **해결 방법**:
  1. 환영 로그를 보낼 대상 채널 권한 설정에 들어가 봇이 '메시지 보내기'와 '링크 첨부' 권한을 갖고 있는지 점검합니다.
  2. 디스코드 개발자 포털(Developer Portal)에서 해당 봇 애플리케이션의 **Privileged Gateway Intents** 설정 중 **`SERVER MEMBERS INTENT`**와 **`MESSAGE CONTENT INTENT`** 항목이 켜져(Enable) 있는지 확인해 주십시오. (이 두 인텐트가 비활성화되어 있으면 초대장 변화 추적이나 메시지 감지가 불가능합니다.)
